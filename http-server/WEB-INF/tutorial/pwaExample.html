PWA Tutorial: Example

Let's build a PWA application that starts a dummy application and checks if the dummy application is finished or not.

Thedummy application is located in UTIL/tutorial directory. This application mimics the actual processing. It will show processing inforation on the console if you execute it from the Command Window.

C:\dev\PWA\http-server\UTIL\tutorial>jjs ExampleApp1.js Processing 1 done. Processing 2 done. Completed Successfully. or C:\dev\PWA\http-server\UTIL\tutorial>jjs ExampleApp1.js Processing 1 done. Processing 2 done. Failed.

The PWA application has "Run" button and one "Status" text area where it shows the processing log like above.

  • This browser-side uses VUE-JS, which consists of one HTML file, exampleApp1.html, and one JavaScripts file, comp-exampleApp1.js.
  • The server-side uses jjs/tutorial/runApp.js.

Click PWA Applicationto open the page. There are three implementations to track the dummy application's processing.

  • The first implementation is not so efficient so that it takes the longest time to finish. (It takes about 25+ seconds.)
  • The second one is a good implementation. (It takes about 15+ seconds.)
  • The third one is the best one since it checks if the dummy application finishes or not in addition to the second implementation.

After processing, please see the OS' Temp directory. You will see the following 4 log files.

  • exampleApp1CmdWinController....txt
  • exampleApp1CmdLogger....txt
  • checkExampleApp1DoneCmdWinController....txt
  • checkExampleApp1DoneCmdLogger....txt
The "exampleApp1" and "checkExampleApp1Done" are the "TempFilePrefix" in the scripts.
